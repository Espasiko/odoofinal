# Integración exitosa de Groq LLM en n8n (20/07/2025)

## Problema inicial
- Error persistente "A Chat Model sub-node must be connected and enabled" en el flujo de trabajo de n8n
- Credenciales incorrectas o mal configuradas para Groq API
- Conexión incorrecta entre nodos AI Agent y Groq Chat Model
- Falta de herramientas conectadas al AI Agent

## Solución implementada
1. **Identificación del problema raíz**:
   - Conexión incorrecta entre nodos: se usaba puerto `llm` en lugar de `ai_languageModel`
   - API key inválida o mal configurada en las credenciales
   - Falta de variable de entorno GROQ_API_KEY en el contenedor Docker
   - Mensaje del sistema no habilitado correctamente en el AI Agent
   - Falta de herramientas conectadas al AI Agent

2. **Creación de credenciales correctas**:
   - Creación de credenciales tipo `groqApi` con la API key de pago (`CLAVE_API_ELIMINADA_POR_SEGURIDAD`)
   - Creación de credenciales tipo `groqApi` con la API key gratuita (`CLAVE_API_ELIMINADA_POR_SEGURIDAD`)
   - Configuración correcta de acceso a nodos para `@n8n/n8n-nodes-langchain.lmChatGroq`

3. **Configuración del flujo de trabajo con API de pago**:
   - Uso del nodo `@n8n/n8n-nodes-langchain.lmChatGroq` versión 1.0
   - Conexión del puerto `ai_languageModel` del nodo Groq Chat Model al AI Agent
   - Configuración del modelo `llama-3.1-8b-instant` con parámetros optimizados

4. **Configuración del flujo de trabajo con API gratuita**:
   - Misma estructura de nodos que el flujo con API de pago
   - Adición de un nodo de herramienta "Información Odoo 18" conectado al puerto `tools` del AI Agent
   - Habilitación explícita del mensaje del sistema con `systemMessageEnabled: true`

5. **Actualización del entorno Docker**:
   - Adición de variable de entorno GROQ_API_KEY en docker-compose.yml
   - Reinicio del contenedor para aplicar los cambios

6. **Validación de la solución**:
   - Flujos de trabajo validados sin errores críticos
   - Conexiones correctas entre nodos
   - Credenciales válidas y funcionando
   - Herramientas conectadas correctamente al AI Agent en el flujo con API gratuita

## Detalles técnicos
- **Nodos utilizados**:
  - Chat Trigger (`@n8n/n8n-nodes-langchain.chatTrigger` v1.2)
  - Groq Chat Model (`@n8n/n8n-nodes-langchain.lmChatGroq` v1.0)
  - AI Agent (`@n8n/n8n-nodes-langchain.agent` v2.2)
  - Set (nodo de herramienta para información sobre Odoo 18)

- **Configuración de conexiones para flujo con API gratuita**:
  ```json
  "connections": {
    "When chat message received": {
      "main": [[{"node":"AI Agent","type":"main","index":0}]]
    },
    "Groq Chat Model": {
      "ai_languageModel": [[{"node":"AI Agent","type":"ai_languageModel","index":0}]]
    },
    "Información Odoo 18": {
      "main": [[{"node":"AI Agent","type":"tools","index":0}]]
    }
  }
  ```

- **Credenciales**:
  - Tipo: `groqApi`
  - API Keys: Configuradas correctamente con claves de pago y gratuita

## Lecciones aprendidas
1. La conexión entre el modelo de chat y el AI Agent debe ser a través del puerto `ai_languageModel`, no `llm`
2. Las credenciales deben estar correctamente configuradas con una API key válida
3. La variable de entorno GROQ_API_KEY debe estar disponible en el contenedor Docker
4. Es importante validar el flujo de trabajo para detectar errores y advertencias
5. Para que el mensaje del sistema funcione correctamente, debe habilitarse explícitamente con `systemMessageEnabled: true`
6. El AI Agent necesita herramientas conectadas para funcionar correctamente y evitar advertencias
